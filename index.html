Data Collection Using OpenStreetMap

I collected data for each zip code in my dataset by using OpenStreetMap to get the nearby amenities by type, counting the distribution of amenities within a few miles of any given location. The below example just shows Irvine's data because I don't want to overload the site, but I collected data for many major cities across the country.

        with concurrent.futures.ThreadPoolExecutor() as executor:
            for city in list(unique_cities):
                if city in city_metadata or type(city) != str:
                    continue
                print(f"Attempting {city}")
                future = executor.submit(fetch_amenities, city)
                try:
                    amenities = future.result(timeout=timeout_threshold)
                    amenities = amenities[amenities.geometry.type == 'Point']
                    amenities['longitude'] = amenities.geometry.x
                    amenities['latitude'] = amenities.geometry.y
                    df_trimmed = amenities.reset_index()[['amenity', 'longitude', 'latitude']]
                    df_trimmed.to_csv(f"geospatial_data/{city}.csv")
                    city_metadata.append(city)
                except concurrent.futures.TimeoutError:
                    break

Feature Extraction

I used the haversine distance formula to get the number of amenities within 50 miles of every zipcode in my dataset.

Formula:

d = 2r * arcsin(√(sin²((lat₂ - lat₁)/2) + cos(lat₁) * cos(lat₂) * sin²((lon₂ - lon₁)/2)))

Where:

  • lat₁ = Latitude of point 1
  • lon₁ = Longitude of point 1
  • lat₂ = Latitude of point 2
  • lon₂ = Longitude of point 2

d = Distance between the two points (along the surface of the sphere)

r = Radius of the Earth (mean radius = 6,371 km)
